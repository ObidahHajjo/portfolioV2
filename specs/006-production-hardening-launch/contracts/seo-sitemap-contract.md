# Contract: SEO Metadata and Sitemap

**Branch**: `006-production-hardening-launch` | **Date**: 2026-03-31

---

## Metadata Resolution Contract

### Input Sources (priority order)

1. Per-page `SeoMetadata` record from PostgreSQL (`pageSlug` match)
2. Global defaults from profile/hero content
3. Environment fallback (`NEXT_PUBLIC_APP_URL` for canonical base)

### Required Output Tags Per Public Page

- `<title>`
- `<meta name="description">`
- `<link rel="canonical">`
- Open Graph: `og:title`, `og:description`, `og:type`, `og:image`, `og:url`
- Twitter Card: `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`

### Fallback Rules

- If `pageTitle` missing -> fallback to deterministic default (page-specific template)
- If `metaDescription` missing -> fallback to deterministic default summary
- If `canonicalUrl` missing -> fallback to `NEXT_PUBLIC_APP_URL + pagePath`
- If `ogTitle`/`ogDescription` missing -> reuse resolved title/description
- If `ogImageUrl` missing -> fallback to default portfolio social image

### Admin Data Contract Impact

Existing admin SEO endpoints remain in use (`/api/admin/seo-metadata` and `/api/admin/seo-metadata/[id]`) with `canonicalUrl` added to payload schema.

## Sitemap Contract

### GET /sitemap.xml

Generated by Next.js metadata route (`src/app/sitemap.ts`).

### Inclusion Rules

Include:

- Stable public routes (`/`, `/case-studies`, etc.)
- Dynamic public content routes where record is `published = true` and `isVisible = true`

Exclude:

- `/admin/*`
- Draft/unpublished content
- Missing/deleted records
- Any route resolving to 404

### Required Output Fields per URL

- `loc` (absolute URL)
- `lastmod` (ISO date/time)

### Example XML Snippet

```xml
<url>
  <loc>https://portfolio.example.com/case-studies/led-platform-migration</loc>
  <lastmod>2026-03-31T08:20:00.000Z</lastmod>
</url>
```

### Failure Behavior

- If dynamic query fails, return sitemap containing only stable static URLs (never 500 for crawler requests).
